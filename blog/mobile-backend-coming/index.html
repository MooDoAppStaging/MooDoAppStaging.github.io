Beta test of new backend
November 16, 2018 | updates

The new backend is ready for testing! You can use the beta of the new version at https://beta.moo.do/w/ and download the new app at https://beta.moo.do/download/. If you want to join the mobile beta, talk to Jay in the beta Slack or email support.

The introduction has an “Import your data” page with an “Import from Moo.do” button which will copy your old account, data, and settings over.

We suggest using the desktop app to reduce any possible chance of data loss because it backs up your data to local files every 10 minutes, and that makes us feel safer.

The Major Changes

Storage and sync

Documents are now stored and synced with Firebase instead of your Drive account. When you load the page it now only gets new changes since the last sync, so the initial sync on load will be much faster. In our testing, the connection seems much more stable than before. And there’s no longer a maximum document size.

Multiple documents

Each pane can now contain its own document. The pane sidebar that was the overview is now doing double duty as a document selector. Click the at the top left of a pane to choose the document for that pane, and click the outline icon on the document to open its overview.

Dates

Dates are now stored separately from the text so that we can add more date features without cluttering the text. You can add a date by pressing + Shift + D. That opens a popup where you can select a date on a calendar or type the date in text. This new system will let us add more date features in the future.

Encryption

All items are encrypted client-side before sending them to the database, and you can add an additional layer of security by using your own encryption password. You will be prompted to (optionally) set a password during the introduction.

Plugin authorization

We totally rewrote the way we authenticate plugins which should fix any sync issues you might have seen before.

Known issues

  • When enabling plugins in the desktop apps, the popup window goes white and doesn’t close, but it did work. If you close the window and reload the app, the plugin will be enabled. This is caused by a bug in Electron which should be fixed in the next update: https://github.com/electron/electron/pull/15216
  • Some items on calendar are showing twice
  • Notifications may not work yet
  • Enabling plugins with multiple accounts causes reauthorize messages for each account, but after reauthorizing everything works
  • Undo/redo may be slightly buggy
  • Google may say the app is not verified when enabling plugins. We submitted for verification but it will take a few days.
